Day 11：应用文写作专题

公学 · 每日推送2026年6月8日Day 11 / 16第二阶段 · 写作专题

13 种高频文体（10 种法定公文 + 3 种事务文书），每种一张格式骨架。今天的目标不是写——是看和记。把每种文种的标题格式、正文结构、落款要求刻在脑子里，上考场之后，一看题干就知道这个文种该怎么搭架子。13 种文体覆盖下行（通知/通报/批复/决定）、上行（请示/报告）、平行（函）、纪要、简报、意见——是《综合知识》最可能考到的写作方向。

每种文种的骨架包含三个必须记住的信息：标题怎么写 / 正文分几段 / 结尾用什么。

今日专题：10 种高频应用文格式全覆盖 | 明日：Day 12 时政专题

一、下行文——4 种

下行 ① 通知

【标题】
XX（发文机关）关于召开XX会议（/开展XX工作/印发XX方案）的通知
【主送】
XX单位、XX部门：（可多点）
【开头】
为……（目的），根据……（依据），现就有关事项通知如下：
【正文】
分条列出：一、事项一 二、事项二 三、事项三。每条写清楚「做什么、怎么做、什么时间完成」
【结尾】
特此通知（或自然收束）
【落款】
发文机关署名 + 成文日期（阿拉伯数字）

致命错误：①标题缺「关于」 ②内容模糊——通知要让人知道「要做什么」 ③把通知和通报搞混——通知=事前布置，通报=事后告知

下行 ② 通报

【标题】
XX关于XXX（情况/表彰/批评）的通报
【主送】
可多点
【开头】
为……（背景），……（何时做了何事）。现将有关情况通报如下：
【正文】
1. 总体情况（好的先说）→ 2. 存在问题（具体、可点名）→ 3. 下一步要求（限期+措施）
【结尾】
特此通报
【落款】
发文机关署名 + 成文日期

致命错误：①只有批评没有「总体情况」的肯定部分——通报要先说好的再说问题 ②问题笼统——必须具体到人/事/数据 ③结尾写了「妥否请批复」——那是请示的结尾

下行 ③ 批复

【标题】
XX关于同意（/不同意/原则同意）XXX的批复
【主送】
来文请示的单位（单一）
【开头】
你单位《关于XXX的请示》（XX〔2026〕X号）收悉。经研究，现批复如下：
【正文】
先表态（同意/不同意/原则同意）→ 再提要求（如同意：须注意什么；如不同意：说明理由或替代方案）
【结尾】
此复（或自然收束）
【落款】
发文机关署名 + 成文日期

致命错误：①没有引用对方来文的标题和文号——批复的第一句必须引用请示 ②标题里没写表态（「关于XXX的批复」→ 缺表态词。应写「同意/不同意/原则同意」）

下行 ④ 决定（表彰型）

【标题】
XX关于表彰XXX的决定
【主送】
可多点或泛称
【开头】
背景 + 事迹概述（1-2句话）——「为表彰先进，树立典型，……决定：」
【正文】
1. 表彰名单及荣誉称号 2. 希望与号召（希望受表彰者……号召大家……）
【结尾】
自然收束（不写「特此决定」）
【落款】
发文机关署名 + 成文日期

二、上行文——2 种

上行 ⑤ 请示

【标题】
XX关于XXX（请求事项）的请示
【主送】
只写一个上级（绝对不多写）
【开头】
请示理由——为什么需要上级批？写客观事实+政策依据+紧迫性。例：「……存在安全隐患，消防部门已下达整改通知……根据XX规定……」
【正文】
请示事项——具体、可操作、数字精确。分项列出：一要钱多少、二要批什么。如果涉及经费：逐项列出费用构成
【结尾】
妥否，请批复（不写「请批示」——批示不是公文用语）
【落款】
署名 + 日期 + 附注：联系人姓名和电话

致命错误：①一文多事 ②主送两个上级 ③结尾写成「妥否请批示」 ④缺附注联系人电话 ⑤标题写成了「报告」

上行 ⑥ 报告

【标题】
XX关于XXX（工作情况/检查情况/整改情况）的报告
【主送】
上级机关（通常一个）
【开头】
报告的目的和依据——「根据……要求/安排，……现将有关情况报告如下：」
【正文】
一、基本情况 二、主要做法或检查结果 三、存在问题 四、下一步措施（或整改情况）
【结尾】
特此报告（绝不写「妥否请批复」——报告不要求上级回复）
【落款】
署名 + 日期

致命错误：①结尾夹带「妥否请批复」——标题是「报告」但结尾求批复=必错 ②只有成绩没有问题——工作/检查报告必须有「存在问题」部分 ③笼统写「存在一些问题」——要具体

三、平行文——1 种

平行 ⑦ 函

【标题】
XX关于商请（/询问/告知）XXX的函
【主送】
对方单位（点对点）
【开头】
说明来意——「为……，现函请贵单位……」语气平等，不卑不亢
【正文】
具体事项 + 理由 + 附件说明。商洽函要写「请贵单位予以支持」；询问函要写「请函告」
【结尾】
「敬请函复为盼」/「特此函告」——绝不写「此致敬礼」
【落款】
署名 + 日期

致命错误：①结尾写「此致敬礼」→ 函不是私人信件 ②用了「妥否请批复」→ 那是请示的结尾 ③语气像命令下级——函是平行关系，要客气

四、纪要 + 事务文书——2 种

下行 ⑧ 纪要

【标题】
XX纪要（不加「会议」二字）
【开头】
时间 + 地点 + 主持人/召集人 + 会议名称 + 参会单位——五要素。「2026年X月X日，XX在XX会议室主持召开XX会议。XX、XX等单位负责同志参加。现将会议情况纪要如下：」
【正文】
三段式——一、会议听取了什么 二、会议讨论了/分析了什么 三、会议决定/部署了什么（核心=议定事项，不是流水账，是结论和分工）
【结尾】
自然收束，不写「特此纪要」
【落款】
署名 + 日期。一般不盖章

事务 ⑨ 简报

【版头】
简报名称 + 第X期 + 编发单位 + 日期（四要素）
【标题】
提炼主题——简明有力的一句话
【正文】
导语式开头（1-2句交代背景/目的/总体成效）→ 分条列出做了什么（每条配数据）→ 最后一句：下一步打算
【版记】
报：XX（上级） / 发：XX（下级和相关单位）

致命错误：①没有数据——简报的生命是数据。每条必须有数字 ②没有「报/发」范围 ③正文写成流水账——简报要有提炼

五、多向行文——1 种

多向 ⑩ 意见

【标题】
XX关于XXX（促进/推进/加强XXX工作）的意见
【主送】
根据方向定——下行意见写下级、上行意见写上级、平行意见写平级
【开头】
目的+依据——「为……，根据……，现提出如下意见：」
【正文】
分条列出见解和办法——每条=「是什么+为什么+怎么做」。可以分大标题+小条目
【结尾】
自然收束或「以上意见，请结合实际认真贯彻落实」（下行）/「以上意见，供参考」（上行或平行）
【落款】
署名 + 日期

致命错误：①意见当成了请示或通知来写——意见重在「提出见解和办法」，不是请示也不是命令 ②只有看法没有具体办法——意见必须有「怎么做的建议」

六、事务文书补充——3 种非公文但常考

⑪ 计划（工作方案）

【标题】
XX单位2026年XXX工作计划（/XXX工作方案）
【开头】
指导思想或工作目标（1段）——「为……，结合……实际，制定本计划。」
【正文】
① 目标任务（量化）→ ② 实施步骤（分阶段，每阶段做什么）→ ③ 保障措施（组织/经费/制度）
【结尾】
落款：制定单位 + 日期

⑫ 总结

【标题】
XX单位2025年XXX工作总结
【开头】
总体概述——「2025年，……取得了……成效。现将全年工作总结如下：」
【正文】
① 主要成绩与做法（数据说话）→ ② 存在的问题与不足 → ③ 经验体会 → ④ 下一年工作打算
【结尾】
落款 + 日期

⑬ 调查报告

【标题】
关于XXX（问题/现象）的调查报告（也可用双标题：主标题揭示主题+副标题「——关于XXX的调查」）
【开头】
调查背景——何时/何地/对谁/用什么方法/为了什么目的做了调查
【正文】
① 基本情况（摆事实+数据）→ ② 问题分析（原因+影响）→ ③ 对策建议（每条建议=具体+可操作）
【结尾】
落款：调查单位/调查人 + 日期

七、万能标题公式与结尾速查

标题万能公式

发文机关 + 关于 + 动词（召开/开展/申请/报送/表彰/推进）+ 事由 + 的 + 文种

把这个公式拆开，任何应用文的标题都能套进去。只需要填两个空：动词和事由。

  • 通知→召开/开展/印发：XX关于召开XX会议的通知
  • 请示→申请/请求：XX关于申请XX经费的请示
  • 报告→汇报/报送：XX关于XX工作情况的报告
  • 通报→表彰/批评/情况：XX关于XX检查情况的通报
  • 批复→同意/不同意：XX关于同意XX的批复
  • 函→商请/询问/告知：XX关于商请XX的函

10 种结尾用语速查

文种标准结尾绝对不能用什么
通知特此通知妥否请批复 / 此致敬礼
通报特此通报妥否请批复
批复此复特此通知 / 特此报告
决定自然收束（不写套话）——
请示妥否，请批复请批示 / 特此报告 / 此致敬礼
报告特此报告妥否请批复 / 请审批
函敬请函复为盼 / 特此函告此致敬礼 / 妥否请批复
纪要自然收束特此纪要
简报自然收束 + 报/发——
意见请结合实际贯彻落实（下行）/ 供参考（上行或平行）妥否请批复
计划自然收束 + 落款——
总结自然收束 + 落款——
调查报告自然收束 + 落款——

⚡ 四种结尾互换是作文题的最高频扣分点：请示=妥否请批复 / 报告=特此报告 / 通知=特此通知 / 函=敬请函复为盼。四个文种的结尾互不通用。在改错题里，只要看到结尾与文种不匹配——直接判错。

⚡ 考前倒计时：这 10 种文种里，请示、报告、通知、函是出题概率最高的 4 种——它们的结尾千万不能写串。请示=「妥否请批复」、报告=「特此报告」、通知=「特此通知」、函=「敬请函复为盼」。——四种结尾，四种文种，换一个就错。
